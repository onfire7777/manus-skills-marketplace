# Design Review & Feedback

Purpose: Use this file when running design critique, structuring feedback, or screening review anti-patterns.

Contents:
- review anti-patterns
- three forms of feedback
- review cadence and format
- creative-direction rules

## Review Anti-patterns (`DR-01` to `DR-07`)

| ID | Anti-pattern | Risk | Response |
|----|--------------|------|----------|
| `DR-01` | review without goals | subjective debate | share goals and target user first |
| `DR-02` | vague feedback | unusable direction | name the element, reason, and objective |
| `DR-03` | micromanagement | low autonomy and morale | set goals, not pixel commands |
| `DR-04` | negative-only feedback | defensive posture | note strengths and improvements |
| `DR-05` | solution-first review | root cause missed | understand the problem first |
| `DR-06` | scope drift during review | wasted time | define scope in advance |
| `DR-07` | no response from stakeholders | stalled decisions | use review deadlines and default decision rules |

## Three Forms of Feedback

| Type | Use | Caution |
|------|-----|---------|
| `Reaction` | quick gut response | useful only as a small opening signal |
| `Direction` | clarify the desired shift | ask "why" to uncover intent |
| `Critique` | evaluate against goals | should dominate the review |

Recommended balance:
- `Reaction`: `5%`
- `Direction`: `15%`
- `Critique`: `80%`

## Review Format

Before review:
- share scope and agenda
- share the work in advance
- clarify what is in and out of scope

During review:
- start with context, goals, and user needs
- explain the design rationale
- collect feedback in an ordered format
- reframe subjective comments into goal-based critique
- record decisions in a shared document

After review:
- assign action items, owner, and due date
- distribute notes
- schedule the next checkpoint if needed

Suggested cadence:
- `30 minutes` per week
- `3-7` participants
- rotating facilitator

## Creative-Direction Rules

- `Why-First`: explain goals and rationale before visuals
- `Options, Not Orders`: present `3+` directions with trade-offs
- `Constraints as Creativity`: make constraints explicit before ideation
- `Evidence-Based`: distinguish preference from validated impact
- `Delegation with Context`: pass `why` and `what`, not step-by-step micromanagement
